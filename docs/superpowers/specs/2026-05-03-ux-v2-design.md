# UX v2 — Creator-Friendly Redesign

**Status:** Design approved, ready for implementation plan
**Date:** 2026-05-03

## Goal

Produce a second version of the Streamlit UI inside a new `v2/` folder. The original app (`app.py`, `ui/`) is **not modified**. Both versions run independently:

- Original: `streamlit run app.py`
- New: `streamlit run v2/app.py`

The `v2/` version imports `processing/` from the project root unchanged — only the UI layer is redesigned.

## Target User

Non-technical content creator making Instagram Reels. They care about the visual result, not about Canny thresholds or epsilon ratios.

## Folder Structure

```
blob_tracker/
├── app.py                   ← original, untouched
├── ui/                      ← original, untouched
├── processing/              ← shared, untouched
├── presets.json             ← shared, read by both versions
└── v2/
    ├── app.py               ← new Streamlit entrypoint
    └── ui/
        ├── __init__.py
        └── sections.py      ← redesigned parameter panel
```

`v2/app.py` adds the project root to `sys.path` so it can `import processing` and `import ui` (from root) is avoided — it uses `v2/ui/` only.

## Language

All UI labels are in **Russian**.

## UI Structure

### Left column (unchanged from v1)

- File uploader: «Перетащи видео сюда или нажми для выбора»
- Looping preview player (`st.video`)
- Active layers indicator: small colored pills below the player showing which layers are currently enabled (e.g. `✏️ Контур`, `📦 Рамки`)
- Export button: «⬇ Экспортировать MP4 в полном качестве»

### Right column — parameter panel (`v2/ui/sections.py`)

#### 1. Стиль (Presets)

Row of 5 buttons. Labels renamed to Russian single-word names:

| Original       | v2 label   |
|----------------|------------|

| Blob Fill      | Заливка    |

Clicking writes `st.session_state["params"]` and reruns (same logic as v1).

#### 2. Детекция (always visible, no toggle)

**Чувствительность** — 3 buttons in a row:

Clicking a sensitivity button writes only these three values into `p["detection"]` and reruns. The sliders below immediately reflect the new values. The active button is highlighted.

**Тонкая настройка** — sliders with Russian labels (always visible for fine-tuning):

| Widget                    | Param key      | Range          |
|---------------------------|----------------|----------------|
| Нижний порог контуров     | canny_low      | 0–255          |

The crop_offset slider uses left/right tick labels: «← левее» and «правее →» displayed as caption below the slider.

#### 3. Слои — 5 expander sections

Each section has a header showing the layer name + ВКЛ/ВЫКЛ badge. Enabled sections are expanded by default; disabled sections are collapsed.

In each section, only the most important parameters are shown:

| Section        | Parameters shown in v2                            |
|----------------|---------------------------------------------------|
| ✏️ Контур      | toggle, color picker, thickness slider            |
| 🎨 Заливка     | toggle, color picker, opacity slider              |

#### 5. Footer

Single button: «🔄 Сбросить настройки» → resets to Нeon preset (same as v1 default).

## Active Layers Indicator

Below the preview player, render a horizontal row of pills using `st.markdown` or `st.columns`. Each enabled layer gets a colored pill with its icon and name. Disabled layers are omitted. This gives the user a glanceable summary of what's active without scrolling the parameter panel.

Example:
```
[✏️ Контур]  [📦 Рамки]  [🏷 Подписи]
```

Implementation: iterate `params` keys, check `enabled` flag, render colored `st.markdown` badges inline.

## Sensitivity Button State

The active sensitivity level is indicated by a highlighted button style. Since Streamlit buttons have no native "active" state, use `st.session_state["sensitivity_level"]` to track which level is selected and conditionally apply `type="primary"` to the active button only.

When the user manually moves a slider (canny_low, canny_high, blur_kernel), `sensitivity_level` is set to `"custom"` and all three buttons appear unselected.

## Import Strategy for v2/app.py

```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).parent.parent))

from processing.preview import build_preview_clip
from processing.pipeline import detect_blobs_for_clip, render_clip
from processing.export import export_video
from processing.presets import load_presets
from v2.ui.sections import render_param_panel   # or relative: from ui.sections import ...
```

Since `v2/app.py` is run as `streamlit run v2/app.py`, the CWD is the project root. Adding `parent.parent` to sys.path is sufficient for `import processing` to work. The `v2/ui/sections.py` is imported as a local module.

## Out of Scope for v2

- Any changes to `processing/` modules
- Visual thumbnail previews on preset buttons
- Drag-and-drop reordering of layers
- Real-time partial preview while rendering
- Keyboard shortcuts
