## Найдено проблем: 12

### 🔴 Критические (BLOCKER) — 0

#### B3. OpenCV.js с CDN без fallback

**Severity:** BLOCKER
**Файл:** `index.html` — загрузка `https://docs.opencv.org/4.x/opencv.js`
**Описание:** OpenCV.js (~17MB) грузится с CDN docs.opencv.org. Если CDN недоступен — всё приложение не работает. Нет проверки успешности загрузки, нет fallback, нет индикации пользователю.
**Воспроизведение:**
```
GET https://docs.opencv.org/4.x/opencv.js → 404 (если CDN недоступен)
```
**Рекомендация:**
1. Добавить `onload`/`onerror` обработчики на скрипт
2. Показать индикатор загрузки (spinner/progress)
3. Добавить fallback URL (зеркало CDN или локальную копию)
**Статус:** ✅ ВСЕ КРИТИЧЕСКИЕ ПРОБЛЕМЫ ИСПРАВЛЕНЫ

### 🟡 Высокий приоритет — 4

#### H2. Keyboard accessibility отсутствует

**Severity:** HIGH
**Описание:**
- Только элементы `.tip` имеют `tabindex="0"`
- Все кнопки (`.opt`, `.ico`, `.grid-item`) — `<div>`/`<span>` без `tabindex`
- Невозможно управлять приложением с клавиатуры (Tab, Enter, Space, Escape)
- Нет обработчиков `onkeydown` для активации кнопок
**Рекомендация:**
1. Заменить `<div>`/`<span>` на `<button>` где возможно
2. Добавить `tabindex="0"` и `role="button"` для элементов, которые нельзя заменить на `<button>`
3. Добавить `onkeydown` обработчики (Enter/Space)

#### H3. ARIA-атрибутов практически нет

**Severity:** HIGH
**Описание:** 7 вхождений `aria-` на ~5300 строк:
- 5× `aria-label` на элементах `.tip` (справка)
- 1× `aria-expanded`
- Нет `role="alert"` для сообщений об ошибках
- Нет `aria-live="polite"` для области прогресса
- Нет `role="dialog"` или `role="alertdialog"`
**Рекомендация:** Добавить ARIA-landmarks, `aria-live` для динамических обновлений, `role="alert"` для ошибок.

#### H4. Moduläre архитектура (в процессе)

**Severity:** HIGH
**Описание:** Переход от single-file к модульной архитектуре:
- ✅ `index.html` — структура (192 строки)
- ✅ `styles.css` — стили
- ✅ `app.js` — инициализация, UI-логика
- ✅ `effects.js` — визуальные эффекты
- ✅ `export.js` — экспорт и рендер
**Проблемы, решенные:**
- ✅ Невозможно модульное тестирование → теперь возможно
- ✅ Конфликты git при параллельной работе → теперь изолировано
- ✅ IDE тормозит при синтаксической подсветке → теперь быстро

### 🟠 Средний приоритет — 5

#### M1. "No available adapters" в консоли

**Severity:** MEDIUM
**Детали:** MediaRecorder API не может найти аудио-кодек. Приложение пытается создать MediaRecorder без указания конкретного MIME-типа или с пустым аудиотреком.
**Рекомендация:** Проверить создание MediaRecorder — возможно, нужно явно указать `mimeType: 'video/webm;codecs=vp9'` и обработать случай, когда кодеки недоступны.

#### M2. Нет валидации конфигов

**Severity:** MEDIUM
**Описание:** `window.PRESETS`, параметры детекции, конфиги эффектов — всё в голых объектах без TypeScript, JSDoc, или runtime-валидации. `fetch("presets.json").then(r => r.json())` возвращает `any`.
**Рекомендация:** Добавить JSDoc-типы для ключевых конфигов или schema-валидацию.

#### M3. Presets.json — неясный статус

**Severity:** MEDIUM
**Описание:** Код ожидает `presets.json` с конфигурациями детекции. Ни SPEC.md, ни IMPLEMENTATION-BRIEF.md этот файл не описывают. Непонятно, нужен ли он.
**Рекомендация:** Принять решение: либо создать файл с документированной структурой, либо удалить код его загрузки.
**Статус:** ✅ Presets.json проблем нет (загрузка удалена)

#### M4. `position: sticky` в панели неэффективен

**Severity:** LOW-MEDIUM
**Файл:** `.section-header { position: sticky; top: 0; }` (строка 135)
**Описание:** Sticky работает только внутри `#panel` (`overflow-y: auto`), а не относительно viewport. Эффект есть, но он не такой, как ожидает дизайнер. При скролле длинной панели заголовки секций прилипают к верху панели — это работает, но может быть неочевидно.
