## Что мы строим

**blob_tracker online** — браузерный VFX-инструмент для обработки одного видео
контурными, трекинговыми и генеративными эффектами.

Основной сценарий:

1. Пользователь открывает сайт на компьютере.
2. Загружает видео со своего устройства.
3. Настраивает детекцию и сразу видит результат на выбранном кадре или во время
   воспроизведения.
4. Выбирает визуальный эффект и параметры.
5. Проверяет результат в превью, ставит видео на паузу и перемещается по таймлайну.
6. Выбирает формат и частоту кадров.
7. Экспортирует и скачивает готовое видео.

Видео обрабатывается в браузере пользователя и не загружается на сервер
проекта.

В духе проектов вроде [Space Type Generator](https://spacetypegenerator.com/),
[BOIDS Tool](https://antlii.work/BOIDS-Tool),
[Endless Tools](https://endlesstools.io/) и [3D Gif Maker](https://www.3dgifmaker.com/) —
но для видео-эффектов.

Пользователь сразу попадает в рабочий инструмент, а не в сложный монтажный
редактор или маркетинговый лендинг.

## Архитектура

**blob_tracker online** использует модульную архитектуру для обеспечения:

- **Модульное тестирование** - каждый компонент можно тестировать отдельно
- **Параллельная разработка** - разработчики могут работать над разными файлами одновременно
- **Легкость навигации** - легко найти и отредактировать конкретную функциональность
- **Производительность IDE** - быстрая синтаксическая подсветка и автодополнение

### Файлы проекта

- `index.html` — чистый HTML (~451 строка) с DOM-структурой, CDN importmap и системными модальными окнами
- `styles.css` — CSS стили (~293 строки), дизайн-система, типографика
- `app.js` — ядро приложения (~1561 строка): инициализация, UI-логика, управление состоянием, пайплайн детекции
- `effects.js` — визуальные эффекты (~546 строк): 24 эффекта + 13 пост-эффектов + аудио-модуляция
- `export.js` — экспорт и рендер (~409 строк): Mediabunny MP4/WebM, нативные fallback'и, аудио-пасс-through

### Зависимости

- OpenCV.js (CDN) — контурное обнаружение
- Mediabunny (CDN) — экспорт MP4/WebM с аудио

## Для кого

Основной пользователь — моушн-дизайнер, digital-художник или VJ, которому
нужен быстрый способ получить выразительный эффект без установки
профессионального видеоредактора и без написания кода.

Инструмент должен оставаться понятным человеку без технического опыта, но
решения по интерфейсу и возможностям в первую очередь принимаются для
творческой работы с видео.

### Принципы продукта

- Сразу к работе: загрузка видео является первым действием.
- Параметры объясняются видимым результатом в превью.
- Один понятный процесс: загрузка → настройка → запуск детекции → просмотр → экспорт.
- Исходное видео не отправляется на сервер проекта.
- Неподтверждённые режимы не выдаются за гарантированно поддерживаемые.
- Регистрация, облачное хранение и мобильная версия не обязательны.

## Где будет работать

- Desktop — единственный обязательный класс устройств текущей версии.

- GitHub Pages рассматривается как будущий статический хостинг, но не должен
  включаться до отдельной команды автора.

## Что внутри (устройство проекта)

Проект использует **модульную архитектуру** для максимальной удобности разработки
и поддержки. Код разделён на несколько файлов по функциональному назначению:

- **`index.html`** — чистый HTML с DOM-структурой и системными модальными окнами
- **`styles.css`** — все CSS-стили (цвета, шрифты, расположение элементов)
- **`app.js`** — ядро приложения: инициализация, UI-логика, управление состоянием
- **`effects.js`** — 24 визуальных эффекта + 13 пост-эффектов
- **`export.js`** — экспорт и рендер видео (MP4, WebM, аудио)

Модульная архитектура выбрана для:

- **GitHub Pages** — после разрешения автора возможен статический хостинг
- **Без сервера** — никаких баз данных, бэкендов, регистраций
- **Тестирование** — каждый модуль можно тестировать отдельно
- **Удобство разработки** — легко найти и изменить нужный код
- **Git-совместимость** — параллельная работа без конфликтов

Приложение не полностью автономно: OpenCV.js и Mediabunny загружаются со
сторонних CDN. Поэтому точная формулировка приватности следующая:

> Исходное видео обрабатывается локально в браузере и не отправляется на сервер
> blob_tracker. Для запуска приложение загружает программные библиотеки со
> сторонних CDN.

Без интернета и без заранее сохранённых библиотек первый запуск не
гарантируется.

## Текущий подтверждённый статус

На 24 июня 2026 года существуют:

- загрузка одного локального видео с проверкой формата, размера (≤500 MB) и длительности;
- явный запуск первой детекции после выбора настроек; загрузка видео только готовит превью;
- OpenCV.js-детекция контуров;
- автоматическое воспроизведение превью после детекции;
- пауза, воспроизведение и перемотка по таймлайну;
- live-проба параметров детекции на текущем кадре без подмены полной детекции;
- повторная полная детекция;
- 11 детекторов (edge, motion-diff, color-hsv, contour-area, simple-blob, circles, DoG, flow, accumulation, K-means, watershed);
- 24 визуальных эффекта + 13 пост-эффектов (полный порт Apolotary);
- **аудио-реактивность** — RMS, kick, high, onset detection через Web Audio API;
- аудио-модуляция 12 эффектов + 4 пост-эффектов;
- выбор Source / 30 / 60 FPS;
- экспорт только в исходной ширине, высоте и пропорции загруженного видео;
- выбор MP4 / WebM;
- прогресс и отмена детекции и экспорта;
- Mediabunny 1.49.0 и автоматический выбор кодека;
- экспорт MP4 и WebM с аудио (AAC/Opus) и без аудио;
- полный тестовый набор: 7 сценариев, 50 ассертов, всё пройдено;
- **разные сообщения об ошибках** для видео, аудио, кодека, CDN и памяти;
- **русский и английский интерфейс** — переключение языка в футере панели;
- **About-диалог** — встроенное описание возможностей, приватности и ограничений.

### Mediabunny: статус

Экспорт MP4 и WebM работает с аудио и без аудио. Подтверждено:

- Mediabunny и OpenCV.js загружаются;
- для MP4 выбирается AVC/H.264, для WebM — VP9;
- MP4 с AAC-аудио проходит автоматическую проверку (ffprobe: аудио есть, длительность ±0.05s, полный декод без ошибок);
- WebM с Opus-аудио проходит автоматическую проверку;
- видео без аудиодорожки экспортируется в MP4 и WebM;
- отмена останавливает процесс и возвращает интерфейс в норму;
- нет утечек AudioSample.

### Экспортный контракт

Экспорт не является пиксель-в-пиксель копией исходника: видео заново рисуется
на Canvas, эффект накладывается, а видео и аудио заново кодируются.

Перед публикацией должны выполняться условия:

- исходные ширина/высота/соотношение сторон и выбранный FPS соблюдены;
- длительность отличается от исходника не более чем на один выходной кадр;
- аудио присутствует и синхронизировано, если оно было в исходнике;
- готовый файл полностью декодируется без ошибок;
- невозможность сохранить аудио показывается как явная ошибка;
- отмена не оставляет зависший интерфейс, незакрытые ресурсы или повреждённую
  загрузку.
